4. Commands-
commit---> version of file synchronization with changes 
    a. git init -  initialize empty Git repository
    b. git status - check type of branch we are on 
    c. git add
    d. git commit -m "{comments}"   --- commit in master branch
    e. git config --global [user-name/email-address]
    f. git log  --- commits, files , works today etc.
    g. git checkout<commit.id / branch-name> opens the branch with its versio of code (generally set to master)
    h. git branch <branchname>
    i. git remote add origin "{URL of repository}"
    j. git remote --v 
    k. git push origin master
    l. git clone <repo URL>
    m. code <name>  - open code

    a. vim ____.txt -- create a file with specified name and add text through terminal itself

        then :q-- quit without saving  and  :w- write this into file and   :wq- write this and quit this mode 
    
    b. press i to enter the the file through the terminal and add the necessary commnds 
    c. to forcefully exit the vim text without changing the file or not saving the file :q !  ( ! is used for override )
